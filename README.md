# 🚀 Стартап-идея: Приложение для поиска единомышленников


# Шаг 1. Входная фиксация сегмента и сценария

## Сегмент
Молодые люди **18–30 лет**, проживающие в городах России, активно использующие мобильные приложения и заинтересованные в совместных активностях (игры, спорт, прогулки, хобби).

## Сценарий использования
Пользователь хочет поиграть в настольную игру, онлайн-игру или заняться спортом, но у него нет компании. Он открывает приложение, выбирает интересующую активность и находит людей рядом, которые хотят присоединиться.

## Основная проблема

- людям сложно найти компанию для совместных активностей  
- существующие друзья не всегда свободны  
- поиск людей через чаты и форумы занимает много времени  

## Последствия

- отказ от активности  
- снижение социальной активности  
- проведение свободного времени в одиночку  

## Текущая замещающая альтернатива

Сейчас люди решают проблему через:

- чаты Telegram / Discord  
- знакомых и друзей  
- форумы и игровые сообщества  
- приложения знакомств

## Окна возможностей

Где конкуренты слабы:

- нет **быстрого поиска людей рядом**
- нет **приложения именно для совместных активностей**
- поиск занимает **много времени**
- нет **единой платформы для разных видов активностей**


# Шаг 3. Критерии выбора сегмента

Основные критерии выбора решения:

1. **Скорость поиска компании** ⭐  
2. **Наличие людей рядом (геолокация)** ⭐  
3. **Безопасность и доверие** ⭐  
4. Простота использования приложения  
5. Популярность сервиса (количество пользователей)  
6. Бесплатность или низкая цена  
7. Возможность фильтрации по интересам  

**Топ-3 критериев**

- скорость поиска  
- геолокация  
- безопасность


# Шаг 4. Гипотеза ценностного предложения

## Вариант 1

Для молодых людей 18–30 лет, которые хотят найти компанию для игр или активностей,  
мы обеспечиваем быстрый поиск людей рядом,  
улучшая скорость организации встречи по сравнению с чатами и форумами,  
за счёт геолокации и алгоритма подбора по интересам.

## Вариант 2

Для пользователей, которые хотят найти компанию для настольных или онлайн-игр,  
мы создаём платформу поиска игроков,  
улучшая удобство и скорость поиска по сравнению с Discord и форумами,  
за счёт автоматического подбора и создания игровых событий.

## Вариант 3

Для молодых людей, ищущих новые знакомства и совместные активности,  
мы предоставляем сервис поиска людей по интересам,  
улучшая вероятность найти подходящую компанию по сравнению с социальными сетями,  
за счёт фильтрации интересов и геолокации.

## Основная гипотеза

Для молодых людей 18–30 лет, которые хотят найти компанию для игр или активностей,  
мы обеспечиваем быстрый поиск людей рядом,  
улучшая скорость организации встречи по сравнению с чатами и форумами,  
за счёт геолокации и алгоритма подбора по интересам.

### Отличие от конкурентов

- поиск людей **по интересам и геолокации**
- быстрый подбор компании
- единая платформа для разных активностей

### Почему сегмент должен поверить

- простое мобильное приложение  
- удобный интерфейс поиска активностей  
- система профилей и рейтингов пользователей  


# Шаг 5. Определение ключевого функционала (ядро MVP)

## Core (ядро MVP)

1. Регистрация и профиль пользователя  
2. Выбор интересов (игры, спорт, хобби)  
3. Поиск людей рядом  
4. Создание активности (игра, встреча)  
5. Присоединение к активности  
6. Чат между участниками  

## Support

- фильтры по интересам  
- рейтинг пользователей  
- отзывы после встречи  
- push-уведомления  
- карта активностей  

## Later

- система рекомендаций  
- интеграция с игровыми платформами  
- подписка и премиум функции  
- мероприятия и турниры  
- система достижений  


# Шаг 6. Гипотезы проверки ценности

## Гипотеза 1
Пользователи испытывают проблему поиска компании для активностей.

Метод проверки:  
интервью с 15–20 респондентами.

Критерий:  
≥70% подтверждают проблему.


## Гипотеза 2
Пользователи готовы использовать приложение для поиска компании.

Метод проверки:  
демонстрация прототипа.

Критерий:  
≥60% готовы попробовать сервис.


## Гипотеза 3
Пользователи готовы оставить контакт для доступа к приложению.

Метод проверки:  
лендинг с описанием сервиса.

Критерий:  
конверсия ≥10%.


## Гипотеза 4
Функция поиска людей рядом является ключевой ценностью.

Метод проверки:  
опрос пользователей.

Критерий:  
≥50% называют её основной функцией.
